    **DESG-05: Heatmap keyboard grid navigation (per CONTEXT.md locked decisions)**

    In ConfigHeatmap.svelte:
    1. Change the SVG `role` from `"img"` to `"grid"` when `interactive` prop is true (keep `role="img"` for non-interactive scroll reveal usage). Add `tabindex="0"` when interactive so the SVG is keyboard-focusable.
    2. Add a `focusedCell` state: `$state<{ precisionIdx: number; batchIdx: number } | null>(null)`.
    3. Add `onkeydown={handleKeyNavigation}` handler on the SVG element:
       - ArrowRight: move precisionIdx +1 (clamp to max)
       - ArrowLeft: move precisionIdx -1 (clamp to 0)
       - ArrowDown: move batchIdx +1 (clamp to max)
       - ArrowUp: move batchIdx -1 (clamp to 0)
       - Enter / Space: trigger cell selection (same as click) via existing onCellClick callback
       - Escape: clear selection
       - All arrow keys call `e.preventDefault()` to prevent page scroll
    4. Add a visual focus indicator: when `focusedCell` matches a cell, render a 2px stroke outline (using `--color-focus-ring` token or fallback `#005fcc`) around that cell rect. This is a separate `<rect>` overlay with `pointer-events="none"`.
    5. Add `aria-activedescendant` on the SVG pointing to the focused cell's `id`. Each cell `<g>` gets an `id` like `cell-{precisionIdx}-{batchIdx}`.
    6. Change cell `<g>` elements: `role="gridcell"`, `aria-rowindex={batchIdx + 1}`, `aria-colindex={precisionIdx + 1}`, `aria-selected={isSelectedCell}`.
    7. Add a new callback prop: `onCellFocus?: (cell: HeatmapCell | null) => void` - fires when keyboard focus changes so the parent can update the aria-live region.

    In ConfigHeatmapInteractive.svelte:
    1. Add an `aria-live="polite"` region (visually hidden `<div>`) that announces the focused cell's details when keyboard navigation occurs. Text: "{precision} at batch size {batch_size}: {energy} J/token, {ratioVsBest}x vs optimal".
    2. Wire the new `onCellFocus` callback from ConfigHeatmap to update the aria-live region text.
    3. Ensure the tooltip also appears for keyboard-focused cells (not just mouse hover). When `focusedCell` changes, call the same tooltip logic as `onCellHover`.
